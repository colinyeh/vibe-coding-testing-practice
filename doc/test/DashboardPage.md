# DashboardPage 測試案例

> 狀態：初始為 [ ]、完成為 [x]
> 注意：狀態只能在測試通過後由流程更新。
> 測試類型：前端元素、function 邏輯、Mock API、驗證權限...

---

## [ ] 【前端元素】介面應包含儀表板標題與歡迎訊息
**範例輸入**：成功登入進入 `/dashboard`  
**期待輸出**：頁面顯示「儀表板」標題，及「Welcome, [username] 👋」訊息

---

## [ ] 【前端元素】管理員應看到管理後台連結
**範例輸入**：以 admin 角色登入  
**期待輸出**：導覽列顯示「🛠️ 管理後台」連結

---

## [ ] 【前端元素】一般用戶不應看到管理後台連結
**範例輸入**：以 user 角色登入  
**期待輸出**：導覽列隱藏「🛠️ 管理後台」連結

---

## [ ] 【Mock API】成功載入並顯示商品列表
**範例輸入**：進入頁面，API `/api/products` 回傳成功資料  
**期待輸出**：顯示「商品列表」標題，並以 Grid 形式列出所有商品名稱、描述與價格

---

## [ ] 【Mock API】載入失敗時顯示錯誤容器
**範例輸入**：API `/api/products` 回傳 500 錯誤  
**期待輸出**：顯示「⚠️」圖示及錯誤訊息「無法載入商品資料」或 API 回傳的錯誤訊息

---

## [ ] 【function 邏輯】點擊登出應清除權限並跳轉
**範例輸入**：點擊「登出」按鈕  
**期待輸出**：執行 `logout` 函式，LocalStorage 清除 Token，頁面跳轉至 `/login`
